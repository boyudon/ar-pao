# ระบบ AR ส่อง QR Code → วิดีโอท่าน ผอ. ยืนพูด

WebAR แบบ **Image Tracking** (ติดตามรูปภาพ) ด้วย [MindAR](https://github.com/hiukim/mind-ar-js) +
[A-Frame](https://aframe.io/) — เปิดผ่านเบราว์เซอร์มือถือได้เลย **ไม่ต้องติดตั้งแอป**

**มี 2 โหมด:**
- 🟢 **โหมดหลัก — markerless** (`index.html`): ส่อง QR → เปิดกล้อง → **คลิปท่าน ผอ. เกาะตำแหน่งในห้อง** (ไม่ต้องส่องป้าย)
  หมุน/ส่ายมือถือดูได้รอบทิศ (เกาะโลกจริงด้วยไจโร 3DOF). ถ้าถ่าย ผอ. หน้าฉากเขียว + เปิด `chromaKey` จะเห็นเป็น "ท่าน ผอ. ยืนลอย" ในห้องจริง
  *(ข้อจำกัด: เดินเข้าไปใกล้แล้วใหญ่ขึ้น (6DOF) ทำฟรีไม่ได้บน iPhone — ต้องใช้บริการเสียเงินอย่าง 8th Wall)*
- ⚪ **โหมดส่องป้าย — image tracking** (`ar-marker.html`): ส่อง QR → เล็งกล้องไปที่ป้าย/ภาพ → วิดีโอเกาะติดบนป้าย

## 📁 โครงสร้างไฟล์

```
AR อบจ/
├── index.html        ← หน้า AR หลัก: markerless (QR ชี้มาที่นี่)
├── ar-marker.html    ← หน้า AR แบบส่องป้าย (image tracking)
├── config.js         ← ⭐ ตั้งค่าทุกอย่างที่ไฟล์นี้
├── compiler.html     ← เครื่องมือแปลงรูปป้าย → targets.mind (เฉพาะโหมดส่องป้าย)
├── tools/qr.html     ← เครื่องมือสร้าง QR Code
├── js/
│   ├── markerless.js ← ตัวควบคุมโหมด markerless (กล้อง + คลิปลอย + ตัดฉากเขียว)
│   ├── app.js        ← ตัวควบคุมโหมดส่องป้าย
│   └── chromakey.js  ← คอมโพเนนต์ตัดฉากเขียว (โหมดส่องป้าย)
└── assets/
    ├── director.mp4  ← (ใส่เอง) วิดีโอท่าน ผอ.
    └── targets.mind  ← (เฉพาะโหมดส่องป้าย) สร้างจาก compiler.html
```

## 🚀 ขั้นตอนทำให้ใช้งานได้ (ทำตามลำดับ)

### 1. ถ่ายวิดีโอท่าน ผอ.
- ถ่ายแนวนอนหรือแนวตั้งก็ได้ แต่ให้ตรงกับสัดส่วนป้ายที่จะใช้
- บันทึกเป็น `.mp4` (H.264) ความละเอียด **720p** ขนาดไฟล์ **ไม่เกิน ~20 MB** (โหลดเร็ว)
- ตั้งชื่อ `director.mp4` วางในโฟลเดอร์ `assets/`
- 🟢 *อยากได้แบบ "ตัวท่าน ผอ. ลอยขึ้นมา" (โฮโลแกรม)?* ถ่ายหน้า **ฉากเขียว** แล้วดูข้อ "โหมดกรีนสกรีน" ด้านล่าง

### 2. เตรียมรูปป้าย/โปสเตอร์เป้าหมาย
- ออกแบบโปสเตอร์/ป้าย (เช่น รูปท่าน ผอ. + ข้อความ) ที่จะ **พิมพ์ออกมาตั้งจริง**
- รูปควร **มีลวดลาย/รายละเอียดเยอะ คอนทราสต์สูง** ไม่ใช่พื้นเรียบหรือสมมาตรซ้ำ ๆ (ระบบจะจำได้แม่นกว่า)

### 3. คอมไพล์รูปเป้าหมาย → `targets.mind`
- เปิด **`compiler.html`** ในเบราว์เซอร์ (ดับเบิลคลิกไฟล์ได้เลย)
- ลากรูปป้ายเข้าไป → กด "เริ่มคอมไพล์" → ดาวน์โหลด `targets.mind`
- นำไฟล์ไปวางใน `assets/`

### 4. ขึ้นออนไลน์ด้วย GitHub Pages (ต้องเป็น HTTPS กล้องถึงจะเปิดได้)
```bash
# ในโฟลเดอร์โปรเจกต์
git init
git add .
git commit -m "AR director video"
git branch -M main
# สร้าง repo เปล่าบน github.com ก่อน แล้วใส่ URL ของคุณ
git remote add origin https://github.com/<ชื่อผู้ใช้>/<ชื่อ-repo>.git
git push -u origin main
```
แล้วไปที่ repo บน GitHub → **Settings → Pages** → Source เลือก `main` / `/ (root)` → Save
รอสักครู่จะได้ลิงก์ เช่น `https://<ชื่อผู้ใช้>.github.io/<ชื่อ-repo>/`

### 5. สร้าง QR Code
- เปิด **`tools/qr.html`** → ใส่ลิงก์ GitHub Pages ของคุณ → สร้าง → ดาวน์โหลด PNG
- พิมพ์ QR ติดไว้ **ข้างป้ายเป้าหมาย** พร้อมข้อความ "สแกนเพื่อชมวิดีโอแนะนำ"

### 6. ทดสอบ
เปิดลิงก์บนมือถือ → แตะ "เริ่ม" → อนุญาตกล้อง → เล็งไปที่ป้ายที่พิมพ์ → วิดีโอเด้งขึ้น ✅

## ⚙️ การปรับแต่ง (ไฟล์ `config.js`)

| ค่า | ความหมาย |
|---|---|
| `videoSrc` | ที่อยู่ไฟล์วิดีโอ |
| `videoWidth` | ความกว้างวิดีโอเทียบกับป้าย (1 = เท่าป้าย) |
| `videoRotation` | หมุนวิดีโอ เช่น `"-90 0 0"` ถ้าวางป้ายราบกับโต๊ะ |
| `sound` | `true` = เปิดเสียง |
| `chromaKey` | `true` = ตัดพื้นหลังเขียว |
| `ui.*` | ข้อความ/ชื่อหน่วยงานบนหน้าจอ |

### โหมดกรีนสกรีน (ตัดพื้นหลัง)
ใน `config.js` ตั้ง:
```js
chromaKey: true,
chromaColor: "#00d400",   // ปรับให้ตรงสีฉากจริง
chromaSimilarity: 0.4,    // เพิ่มถ้ายังเหลือขอบเขียว
```

## 🧪 อยากลองก่อนมีไฟล์จริง (Demo)
ยังไม่มี `director.mp4` / `targets.mind` ก็ทดสอบไปป์ไลน์ได้ โดยใช้ตัวอย่างของ MindAR:
1. ใน `config.js` ชั่วคราว:
   ```js
   targetSrc: "https://cdn.jsdelivr.net/gh/hiukim/mind-ar-js@1.2.5/examples/image-tracking/assets/card-example/card.mind",
   videoSrc:  "https://cdn.jsdelivr.net/gh/hiukim/mind-ar-js@1.2.5/examples/image-tracking/assets/card-example/kozyrface.png",
   ```
   *(ภาพการ์ดตัวอย่างดาวน์โหลด/พิมพ์ได้จาก [หน้านี้](https://hiukim.github.io/mind-ar-js-doc/quick-start/overview))*
2. เปิดเว็บแล้วเล็งกล้องไปที่การ์ดตัวอย่างเพื่อทดสอบ
3. ใช้งานจริงค่อยเปลี่ยนกลับเป็นไฟล์ของคุณ

## ❓ แก้ปัญหาที่พบบ่อย
- **iPhone: ต้องใช้ Safari เท่านั้น** → Chrome/แอปอื่นบน iPhone เปิดกล้องได้แต่ระบบ AR ไม่ทำงาน
  (เว็บจะเตือนให้เองถ้าเปิดผิดเบราว์เซอร์). ข่าวดี: การสแกน QR ด้วย **แอปกล้องของ iPhone จะเปิด Safari ให้อัตโนมัติ**
- **เปิดผ่าน LINE/Facebook แล้วกล้องไม่ขึ้น** → in-app browser บล็อกกล้อง ให้แตะ ⋯ → "เปิดในเบราว์เซอร์"
- **อยากดูสาเหตุปัญหาแบบละเอียด** → เปิดลิงก์ต่อท้ายด้วย `?debug=1` (เช่น `…/ar-pao/?debug=1`) จะมีแถบ DEBUG บอกทุกขั้นตอน
- **กล้องไม่เปิด** → ต้องเปิดผ่าน **HTTPS** (GitHub Pages เป็น https อยู่แล้ว) และกด "อนุญาต" กล้อง
- **ส่องป้ายแล้วไม่ขึ้นวิดีโอ** → รูปเป้าหมายรายละเอียดน้อยไป / พิมพ์เล็กไป / แสงน้อย ลองเปลี่ยนรูปที่ลวดลายเยอะขึ้น
- **วิดีโอไม่มีเสียง** → บางมือถือต้องแตะหน้าจอก่อน และตรวจว่าไม่ได้เปิดโหมดปิดเสียง
- **โหลดช้า** → ลดขนาดไฟล์ `director.mp4` ให้เล็กลง
